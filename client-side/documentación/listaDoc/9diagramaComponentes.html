Diagrama de Componentes

A continuación explicaremos la organización del diagrama de componentes de nuestro programa.
Nuestro programa se descompone en cuatro bloques principales:

  1. Web View
  2. Controller
  3. Scrapper
  4. Database

Web View

Este bloque está caracterizado por contener todo lo relacionado con la página web sobre la cual ejecutaremos el programa de web scrapping.

Controller

En este bloque hemos desarrollado la GUI (Graphical User Interface), gracias al disponer de una GUI podemos ejecutar cualquier orden sin tener que entrar en especificaciones de código. Complemento de ayuda para a la ejecución de nuestro programa asequible para cualquier nivel de usuario.

Scrapper

En este bloque es donde se desarrolla toda la estructura interna del funcionamiento de las funciones de nuestro programa. Este bloque incluye todo lo necesario para que sea posible la ejecución y el uso de nuestro programa. Al mismo tiempo incluye todas las relaciones internas que existen entre las funciones, para poder entender como se desarrolla el recorrido de la información y la ejecución interna.

Data Base

Este bloque incluye la base de datos sobre la cual el programa va a poder guardar toda información extraída de la página web sobre el cual lo hemos ejecutado.
