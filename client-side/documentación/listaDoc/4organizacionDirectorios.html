Organización de Directorios

Para la organización de directorios de nuestra app hemos seguido el estandarte convencional sugerido por nuestro mentor.

Nuestro programa se divide en 4 directorios:

Client – Side

En el directorio “client-side” hemos incluido todo lo relacionado con la página web a la cual pensamos ejecutar el programa de web scrapping y donde encontraremos todo el contenido visual dirigido al público consumidor. En este directorio podremos encontrar todos los archivos relacionados con los lenguajes de marcas HTML y CSS, al mismo tiempo que enlaces e imágenes que son de gran utilidad y ayuda para la correcta visualización de los servicios página web.

Domain

En el directorio “domain” podremos ver que su interior está subdividido en otros dos directorios distintos, uno designado con el nombre “src” y el otro como “test”.

src

En el “src” podremos encontrar los archivos necesarios de configuración para poder ejecutar pytest y el archivo que contiene las funciones que ejecutarán nuestro programa. Al mismo tiempo incluimos la conexión com mongo en el archivo llamado "PythonToMongo" ya que lo necesitaremos para guardar la información que nuestro programa recoja en la base de datos.

test

En el directorio “test” podemos encontrar los archivos de configuración para poder ejecutar pytest y también podremos localizar un archivo para cada función de nuestro programa, cada archivo será nombrado como la misma función sobre la que se ejecuta. Cada uno de estos archivos está compuesto por distintos casos test que comprobarán y se asegurarán del correcto funcionamiento de nuestro programa.

Repository

En este directorio podremos encontrar el archivo que hace posibilita la conexión entre nuestro programa y nuestra base de datos donde almacenaremos toda la información extraída de nuestro programa de web scrapping. En su interior también encontraremos otro directorio llamado mongodb, en el cual en su interior podremos encontrar el patrón de esquema que hemos seguido a la hora de crear nuestros documentos JSON y una copia del esquema de validación que hemos definido sobre nuestra base de datos para evitar la inserción de archivos no deseados.

Resources

En el directorio “resoruces” hemos decidido incluir todos los recursos que hemos tenido la oportunidad de crear libremente para facilitar y agilizar el desarrollo de nuestro programa, al mismo tiempo que pensábamos que aumentaría su prestigio. En el se incluyen el documento de python:

Generador Web

Generador Web, en este documento se incluye un auto generados de páginas web, para tener la oportunidad de poder generar nuevas páginas ofreciendo nuevos servicios para nuestra página web, a gran velocidad y eficiencia, manteniendo un absoluto control de los datos que decidimos incluir.
